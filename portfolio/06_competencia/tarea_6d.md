# Tarea 6.D: Uso responsable

## Actividad

Auditoría Git.

## Competencia

Evaluar la información digital, reconocer sesgos y riesgos (B1).

## Momento

Hacia el final del curso, como actividad de refuerzo sobre buenas prácticas en desarrollo/seguridad.

## Indicadores de logro

- Análisis comparativo de casos reales identificando vulnerabilidades o riesgos éticos.

## Desarrollo de la actividad

### Grupal - 60 min

Se presenta a los grupos un proyecto de GitHub que contiene fallos críticos (ej.: una clave API expuesta, o código con
una vulnerabilidad conocida).

Los alumnos deben actuar como "Auditores de Código" y realizar un análisis crítico:

- Identificar el riesgo específico (¿es un fallo de Git?, ¿un error humano?).
- Determinar la causa raíz del riesgo (ej.: falta de revisión, uso inadecuado de variables de entorno).
- Proponer soluciones para prevenirlo en el futuro.

Generarán un informe detallado que justifique su crítica y proponga medidas de mitigación.

## Herramientas y recursos

- Casos prácticos simulados (ej.: fragmentos de código con errores intencionados).
- Moodle.
- Google Docs.
